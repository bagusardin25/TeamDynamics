# God Mode 2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing God Mode bar into an honest, backend-authoritative intervention console with targets, categories, deterministic previews, confirmation, receipts, safe undo, pause/step, and next-turn acknowledgement.

**Architecture:** A focused intervention domain service owns deterministic plans, application, rollback, and public receipts. The existing FastAPI and WebSocket layers expose that state, while the existing React panel consumes it without changing the simulation page layout.

**Tech Stack:** Python 3.11, FastAPI, Pydantic, pytest, Next.js 16, React 19, TypeScript, Node test runner.

## Global Constraints

- Preserve the existing three-column simulation layout.
- Keep custom-command effects category-based; never claim free-form semantic branching.
- Require explicit target and category for every new intervention.
- Undo only the latest reversible intervention before acknowledgement and in the same paused round.
- Keep legacy WebSocket intervention payloads compatible.
- Do not add a database migration; persist audit data through message JSON and Redis simulation state.
